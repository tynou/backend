Проект представляет из себя распределенную бэкенд-систему, разработанную в рамках изучения паттернов микросервисной архитектуры, межсервисного взаимодействия и контейнеризации.

## Архитектура проекта

Проект построен на базе независимых микросервисов, каждый из которых имеет свою зону ответственности и собственное хранилище данных.

![Схема](/diagram.svg)

### Компоненты системы:

- **API Gateway:** Единая точка входа. Маршрутизирует запросы к нужным микросервисам.
- **Auth Service:** Отвечает за регистрацию, генерацию OTP-кодов, выдачу JWT токенов и хранение данных пользователей.
- **Posts Service:** Основная бизнес-логика. Управление постами пользователей.
- **Notification Service:** Сервис, отвечающий за отправку уведомлений на электронную почту.

## Технологический стек

- **Базовые технологии:** .NET, ASP.NET Core Web API
- **Архитектурные паттерны:** Microservices, Clean Architecture, CQRS
- **API Gateway:** Ocelot (с настройкой агрегации Swagger-документации)
- **Внутрисервисная маршрутизация:** MediatR
- **Асинхронное взаимодействие:** RabbitMQ
- **Синхронное взаимодействие:** gRPC
- **Базы данных и кэш:** PostgreSQL, Redis
- **Инфраструктура:** Docker, Docker Compose
- **Валидация:** FluentValidation
